# Justification RGPD — Tableau de bord Éligibilité Crédit

## 1. Contexte
Ce projet est un **exercice académique** réalisé dans le cadre d’une étude de cas pour une néo-banque fictive.  
L’objectif est de construire un tableau de bord de démonstration permettant d’afficher un **score d’éligibilité au crédit** ainsi que des informations descriptives sur un client.

## 2. Données utilisées
- Les données proviennent du **jeu de données public Home Credit Default Risk** (Kaggle).  
- Aucune donnée personnelle réelle (nom, adresse, téléphone, email, numéro de carte) n’est utilisée.  
- Les variables utilisées sont limitées à des informations financières anonymisées et synthétiques (ex. âge, revenu, type de contrat).

## 3. Principes RGPD respectés
- **Minimisation des données** : seules les variables nécessaires au calcul du score sont exploitées.  
- **Anonymisation** : les données ne contiennent pas d’identifiants directs permettant de reconnaître une personne réelle.  
- **Limitation de la finalité** : les données sont utilisées uniquement à des fins **pédagogiques et de démonstration**.  
- **Durée de conservation** : aucune donnée utilisateur n’est conservée au-delà de la session d’utilisation de l’application.  
- **Sécurité** : le modèle et l’API ne stockent pas les données saisies. Les traitements sont effectués en mémoire et détruits après usage.

## 4. Conclusion
Dans ce projet, toutes les données sont **fictives et/ou anonymisées**.  
L’application développée est une **démonstration académique** et ne peut être utilisée en production ni pour des décisions financières réelles.  

Ainsi, les exigences principales du RGPD sont respectées.
